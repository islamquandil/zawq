# Zawq (ذوق) — Flutter

MENA-focused social restaurant review app. Native Flutter (Material 3),
clean architecture, Riverpod state management, full EN/AR localization
with automatic RTL.

## One-time setup

The `android/` and `ios/` platform folders are generated by your Flutter SDK
so they always match your toolchain (Gradle/AGP/Xcode versions). From the
project root run once:

```bash
./setup.sh
# or manually:
flutter create . --org app.zawq --project-name zawq --platforms android,ios
flutter pub get
```

This only adds platform scaffolding — it never touches `lib/`, `test/`, or
`pubspec.yaml` (existing files are always kept).

## Run

```bash
flutter pub get
flutter run          # Android or iOS device/simulator
```

## Architecture

```
lib/
  core/          # theme, localization, shared widgets, utils
  domain/        # entities + abstract repositories (pure Dart)
  data/          # seed datasource + repository implementations
  application/   # Riverpod providers & state notifiers (use-case layer)
  features/      # feature-first presentation: home, map, restaurant,
                 # people, lists, profile
```

- State: `flutter_riverpod` StateNotifiers; UI is fully reactive.
- Localization: hand-rolled `L10n` (EN/MSA Arabic) + `flutter_localizations`
  for Material widgets; locale switch flips the app to RTL automatically.
- Data: an in-memory seed datasource behind repository interfaces — swap
  `data/repositories/*` for API-backed implementations without touching
  presentation.
- Arabic typography: Cairo is bundled in `assets/fonts` (SIL OFL licence
  included), so the app needs no network access on first launch.
- Images are local color placeholders today; when real photo URLs land,
  add `cached_network_image` and replace `PhotoTile` in one place.
- Smoke test: `flutter test` runs `test/widget_test.dart` (splash → shell).
